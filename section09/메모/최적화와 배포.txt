# 1강) 이미지 최적화
다양한 이미지 최적화 기법들
- webp, AVIF 등의 차세대 형식으로 변환하기
- 디바이스 사이즈에 맞는 이미지 불러오기
- Lazy Loading 적용하기
- 블러 이미지 활용하기
- etc..
위 기능들을 원래는 코드로 만들어야하는데
NEXT에서는 내장된 <Image> 컴포넌트를 사용하면 자동 적용해준다

이미지 성능 알아보기 위해 개발자도구 Network 탭에서 img 필터 적용, 캐시 비우기 및 강력 새로고침

## 최적화 없이 쌩으로 가져왔을때 문제
1. Type - jpeg인데 무거움 ~> webp, AVIF로 변환하면 더 가벼워짐
2. 즉시 로딩 - 한 화면에 필요한 이미지들보다 더 많은 이미지들을 불필하게 '동시에' 불러옴
3. Size - 실제 화면에 렌더링될 이미지 크기보다 더 큰 사이즈로 불러옴

alt는 예를들어 이미지가 안나오는 상황에 대체할 텍스트

그냥 외부 url의 이미지를 사용하면 에러
next.config.ts에서 images 설정으로 허용할 도메인 추가해줘야함

## 이미지 최적화 After
1. Type: webp
2. Lazy Loading: 시간대별 라인보면 로딩이 1000ms, 1500ms 나뉘어져있음
                 Time도 먼저 가져올 이미지는 빨라짐 20ms -> 10ms
3. Size: 압도적으로 줄어듦 74.2kb -> 0.2kB

# 2강) 검색 엔진 최적화 (SEO)
'메타 데이터 설정하기'만 강의에서 진행, 나머지는 쉽게 금방함

현재의 index 역할 파일은 app/(with-searchbar)/page.tsx
정적 페이지: const metadata: Metadata
            -> (with-searchbar)/page.tsx
동적 페이지: generateMetadata(): Promise<Metadata>
            -> search/page.tsx
동적 페이지 + API 요청 데이터로 메타 설정
            -> book/[id]/page.tsx
