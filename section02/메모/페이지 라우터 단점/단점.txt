<Page Router의 단점>
1. 페이지별 레이아웃 설정이 번거롭다
2. 데이터 페칭이 페이지 컴포넌트에 집중된다
    getStaticProps, getServerSideProps 함수에서 매번 다음 과정 불편
    1) 백엔드 서버로부터 데이터 불러오기
    2) 페이지 컴포넌트에게 Props로 데이터 전달
3. 불필요한 컴포넌트들도 JS Bundle에 포함된다
    - 예) 서버측에서만 사용되는 컴포넌트들도 클라이언트측 JS Bundle에 포함됨
    - 번들 용량이 커지면서
    -> TTI 느려짐

=> App Router 등장
